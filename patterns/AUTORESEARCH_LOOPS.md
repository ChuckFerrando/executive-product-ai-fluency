# Autoresearch Loops

## Pattern Definition

A structured system for automating research using AI through repeated cycles of prompting, evaluation, and refinement.

## Core Idea

Instead of conducting research once, create a loop that continuously improves outputs.

## Loop Structure

1. Define Question
2. Generate Outputs (AI)
3. Evaluate Results
4. Refine Prompt / Inputs
5. Repeat

## Key Components

- Prompt frameworks
- Evaluation criteria
- Iteration logic
- Tool orchestration

## Example Workflow

1. Define research question
2. Generate multiple AI outputs
3. Compare outputs using criteria
4. Identify strongest patterns
5. Refine prompt and rerun

## Use Cases

- Market research
- Competitive analysis
- feature exploration
- messaging testing

## Design Principles

- run multiple variations
- compare outputs systematically
- define evaluation criteria upfront
- iterate quickly

## Failure Modes

- no evaluation criteria
- single-pass outputs
- over-reliance on first result
- lack of iteration

## Skill Development

To master this pattern:
- learn prompt iteration
- design evaluation systems
- think in loops, not tasks

## Outcome

Transforms research from:

manual activity → automated system
