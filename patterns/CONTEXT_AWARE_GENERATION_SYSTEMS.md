# Context-Aware Generation Systems

## Pattern Definition

Embedding generative AI into products by supplying structured, proprietary context to produce differentiated outputs.

## Core Idea

Do not build generic generation features.

Build systems that combine:
- user data
- product data
- historical context
- generation models

## When to Use

- content generation
- design systems
- reporting
- personalization

## Pattern Structure

1. Input Context
   - user data
   - documents
   - prior outputs

2. Model Execution
   - generation
   - synthesis

3. Output Integration
   - embedded in product workflow
   - actionable for users

## Example Use Cases

### 1. E-commerce
- generate product images from existing catalog data

### 2. CRM
- generate personalized sales materials

### 3. Product Tools
- generate PRD summaries and visualizations

## Design Principles

- feed maximum relevant context
- automate context injection
- embed directly in workflows
- avoid standalone “generate” buttons

## Failure Modes

- generic outputs
- no context integration
- feature-level thinking instead of system-level design

## Outcome

Transforms AI from:

generic generation → differentiated product capability
